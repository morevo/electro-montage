# Туннель — доступ к локальной среде по ссылке

Проброс локального Vite dev-сервера наружу через Cloudflare, чтобы другой человек
видел твою среду в реальном времени по ссылке.

## Запуск (два шага, два окна терминала)

**1. Dev-сервер** (из папки проекта `D:\Projects\test-bonsai`):
```powershell
npm run dev
```

**2. Туннель** (в новом окне терминала, не закрывая первое):
```powershell
.\cloudflared.exe tunnel --url http://localhost:5173
```

В выводе туннеля появится ссылка вида `https://xxx-xxx.trycloudflare.com` —
её отправляешь тому, кому даёшь доступ.

> Сначала dev-сервер, потом туннель. Каждый процесс — в своём окне терминала.

## Отключение

- В каждом окне нажми **`Ctrl + C`**.
- Если окна потерялись — прибей по имени:
```powershell
Stop-Process -Name cloudflared -Force
Stop-Process -Name node -Force
```
> ВНИМАНИЕ: `Stop-Process -Name node` убьёт ВСЕ Node-процессы. Если параллельно
> крутятся другие — закрывай выборочно через диспетчер задач
> (Ctrl+Shift+Esc → найти `cloudflared.exe` / `node.exe`).

## Как пользоваться

| Действие        | Что делать                                              |
|-----------------|---------------------------------------------------------|
| Дать доступ     | Запусти оба процесса → скинь ссылку из вывода туннеля    |
| Правки кода     | Просто сохраняй файлы — у зрителя обновится само (HMR)   |
| Закончил показ  | `Ctrl+C` в окне туннеля (сервер можно оставить)          |
| Новый показ     | Запусти туннель снова → ссылка будет НОВАЯ               |

## Важные нюансы

- **Ссылка меняется** при каждом перезапуске туннеля (бесплатный Cloudflare даёт
  случайный поддомен). Постоянную ссылку можно получить только через Cloudflare-аккаунт.
- **Порт 5173** — стандартный для Vite. Если занят, Vite возьмёт 5174 и т.д. —
  тогда в команде туннеля поменяй порт на тот, что Vite покажет в строке `Local:`.
- `cloudflared.exe` уже лежит в корне проекта — качать заново не нужно.

## Что было настроено (один раз)

В `vite.config.ts` добавлено, иначе Vite 6 блокирует запросы с чужого домена:
```ts
server: {
  host: true,
  allowedHosts: ['.trycloudflare.com'],
},
```
